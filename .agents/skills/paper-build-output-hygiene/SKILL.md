# Paper Build and Output Hygiene

## Purpose
Ensure that AutoLabOS-generated paper outputs and reproducibility bundles are:
- generated by AutoLabOS itself
- not manually substituted by the external coding agent
- organized cleanly under `test/output/`

## Use this skill when
Use this skill when the user asks to:

- validate TeX and PDF generation
- clean up output structure
- prevent unnecessary files/folders
- ensure AutoLabOS generates its own artifacts
- make `test/output/` into a coherent bundle

## Core rule
- Any file that AutoLabOS is supposed to generate must be generated by AutoLabOS itself.
- The external coding agent may fix code, prompts, config, wiring, and runtime behavior.
- The external coding agent must not manually author TeX, PDF, result bundles, or paper artifacts as substitutes.

## Expected output structure
When applicable, expect at least:

- `test/output/README.md`
- `test/output/reproduce/`
- `test/output/results/`
- `test/output/paper/`
- `test/output/artifacts/`

## Required validations
1. whether unnecessary files/folders are being created
2. whether duplicate summaries / result artifacts exist
3. whether canonical runtime paths and operator-facing bundles remain consistent
4. whether TeX generation succeeds
5. whether PDF generation succeeds
6. if PDF generation fails, classify the cause:
   - missing toolchain
   - malformed TeX
   - missing assets / refs / paths
   - runtime wiring issue
7. whether brief-level manuscript constraints (2-column, 8-page main body) are actually reflected in paper generation
8. whether `test/output/` is sufficient for a user to understand or reproduce the run

## Output format
Always produce:

1. current output structure
2. unnecessary file/folder status
3. required artifact coverage
4. TeX status
5. PDF status
6. page-budget compliance status
7. wiring/path changes needed
8. next recommended fix

## Guardrails
- Do not patch over failures by manually generating TeX or PDF.
- Do not break runtime-critical canonical paths blindly in the name of hygiene.
- Prefer consolidating and deduplicating over multiplying bundles.
- Do not accept a structure where users must inspect unrelated folders to understand one run.

## Good completion criteria
This skill is complete when:

- AutoLabOS generates its own required artifacts
- `test/output/` has a coherent structure
- TeX→PDF was actually validated
- unnecessary files/folders were reduced
- brief-level layout/length constraints are reflected in paper outputs
